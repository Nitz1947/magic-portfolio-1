## Przegląd

Refaktoryzacja monolitu e-commerce do architektury mikroserwisów skupionej na modułach płatności, zamówień i powiadomień. Cel: niezależne wdrożenia, lepsza odporność na awarie i skalowanie poziome.

## Kluczowe funkcje

- **Serwis płatności** — Stripe webhooks, idempotencja, retry z dead-letter queue.
- **Serwis zamówień** — saga pattern dla transakcji rozproszonych.
- **Event bus** — RabbitMQ do komunikacji asynchronicznej między serwisami.
- **Observability** — OpenTelemetry, structured logging, health checks.

## Stack technologiczny

- **Node.js** + **TypeScript** + Fastify
- **PostgreSQL** (per service) + **RabbitMQ**
- **Docker** + Kubernetes (EKS)
- **GitHub Actions** — CI/CD z testami integracyjnymi

## Wyniki

- Incydenty produkcyjne: **-60%** w ciągu 6 miesięcy
- Czas recovery (MTTR): z 45 min do **12 min**
- Niezależne deploye: **3–5 tygodniowo** (vs 1 w monolicie)

## Wnioski

Migracja stopniowa (strangler fig pattern) pozwoliła utrzymać ciągłość biznesową. Kluczowe było wczesne ustalenie kontraktów API i schematu eventów między zespołami.
