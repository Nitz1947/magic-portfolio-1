## Przegląd

Case study z 6 lat pracy w ekosystemie Open Tibia. Projekt obejmował prowadzenie serwera na **The Forgotten Server (TFS)**, rozwój custom systemów gry w Lua, administrację bazy MySQL i pełną customizację **myAAC** — CMS używanego przez większość serwerów OT.

To była moja ścieżka od pasji community gaming do profesjonalnego developmentu: od skryptów questów po responsywne panele gracza i integracje web ↔ game server.

## Kluczowe obszary

- **TFS / Lua** — custom vocation system, eventy sezonowe, questy z wieloetapową logiką, anti-abuse i balans ekonomii.
- **myAAC (PHP)** — szablony strony, moduł rejestracji i recover, rankingi, shop z integracją payment, panel admina.
- **MySQL** — schematy pod gracze, itemy, logi transakcji; optymalizacja zapytań pod peak online.
- **Frontend community** — landing pages, news feed, responsywne UI w HTML/CSS/JS zanim przejście na React/Next.js.

## Stack technologiczny

- **TFS** (C++ core + **Lua** scripting)
- **myAAC** — **PHP**, szablony, moduły
- **MySQL** — baza serwera i CMS
- **HTML / CSS / JavaScript** — strona community
- **Linux** — hosting, backupy, monitoring

## Migracja do nowoczesnego stacku

Doświadczenie z TFS/myAAC dało solidną bazę pod dalszy rozwój:

| Obszar OT/TFS | Odpowiednik w web dev |
|---------------|----------------------|
| Lua event callbacks | React hooks, event-driven UI |
| PHP moduły myAAC | Next.js Server Components, API routes |
| MySQL schema design | Prisma, PostgreSQL, migracje |
| Community UX | Product design, onboarding flows |

## Wnioski

Community OT wymaga pełnego ownershipu — od balansu gry po stronę rejestracji. Ta holistyczna perspektywa przekłada się bezpośrednio na full-stack web dev: rozumiem produkt end-to-end, dbam o wydajność pod load i iteruję na feedbacku użytkowników.
